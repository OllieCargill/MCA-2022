The three elements I have chosen to compare between MusicXML and MEI are: dynamics, clef and note.

Dynamics

Music XML

MEI

Dynamics in MEI are represented by the dynam element. This element is used only to represent dynamic markings which are represented by text. For example, forte (f), piano (p) or mezzo-forte (mf). This element cannot be used for dynamics represented in other forms, like a crescendo or diminuendo which are represented graphically. A constraint of this element in MEI is that it must have a specified starting point (if one is not stated, then this is considered a semantic error), using one of the following attributes: startid, tstamp, tstamp.ges or tstamp.real. Whereas an ending point can be specified, but is not required.
